**уншиж танлцсан => Git Policy — Хөгжүүлэлтийн багийн дүрэм**

## Branch дүрэм
1. Онцлог засвар томоохон өөрчлөлт бүрт branch нээнэ 
2. жижиг үсэг - ашиглана
3. хэт ерөнхий нэр ашиглахгүй
4. Branch нэр тодорхой утга бүхий байх ёстой 

## Commit дүрэм
1. Байнга commit хийх нэг том commit биш жижиг жижиг 
2. Нэг commit д хамааралгүй олон утгатай зүйлс байж болохгүй 
3. Юу хийсэн биш яагаад хийсэнээ бичих 

## Pull request дүрэм
1. Pr нэг зориогртой байна
2. Pr нээхэд өөрийгөө assign хийх
3. Feedback ирэхэд шуурхай хариулах 

## Review дүрэм 
1. Pr бүрийг дох хаяж 1хүн хянана
2. хянах үедээ кодийн чанар стандартыг баримтлан хийсэн эсэхийг шалгана
3. томоохон commit хийсэний дараа ахин review huselt ligeene

## Merge durem
1. zuwhun approve bolson ii daraa merge hiine 
2. branch aa main ees hotsrogdolgui tsug awj yvna 

## Rebase hezee zuwshuuruh ve 
pr review boloogui uyd undsen branch tai sync hiihed rebase hereglej bolno gehdee bolgoomjtoi baih heregtei

## Branch hamgaalalt 
main luu shuud push hiih horiotoi zuvhun pr-approve-merge
trainee buddy giin zuwshuurulgui merge hiihgui

## Ci shalgalt
ci aldaa garwal log unshaad aldaag zasna
